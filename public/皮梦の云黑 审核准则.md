# 云端黑名单审核细则（V1）

## 第一章 审核细则目的

### 第一条 制定目的

为统一举报、申诉、复核及黑名单处理过程中的审核标准，降低主观判断差异，提升处理一致性、可追溯性与公平性，特制定本细则。

### 第二条 适用范围

本细则适用于：

1. 用户举报审核；
2. 黑名单等级判定；
3. 黑名单升级、降级、下黑复核；
4. 严重违规事件专项审查；
5. 申诉案件复审。

### 第三条 审核基本原则

审核人员在处理案件时，应遵循以下原则：

1. **先证据，后判断**；
2. **先行为，后态度**；
3. **先事实，后推测**；
4. **先风险，后处罚**；
5. **同类案件尽量同尺度处理**。

---

## 第二章 审核对象与材料要求

### 第四条 审核对象

审核对象包括但不限于：

1. 被举报用户的行为；
2. 被举报用户的历史记录；
3. 举报人提交的证据材料；
4. 管理后台已有的日志、记录、处罚历史；
5. 与案件相关的申诉、补充说明及复核意见。

### 第五条 证据材料类型

可作为审核依据的材料包括：

1. 聊天截图；
2. 原始聊天记录；
3. 后台日志；
4. 录屏；
5. 用户提交的图片、文件、语音记录；
6. 机器人或系统自动记录；
7. 多名管理人员交叉核验结果。

### 第六条 证据有效性判定

证据分为以下三类：

#### （一）强证据

满足以下条件之一的，可视为强证据：

1. 含完整上下文的聊天记录；
2. 可核验时间、对象、行为内容的后台日志；
3. 连续录屏或完整过程记录；
4. 多份独立材料能互相印证。

#### （二）一般证据

存在一定证明力，但仍需结合其他信息判断的材料，例如：

1. 单张截图但上下文不完整；
2. 仅有部分聊天内容；
3. 来源真实但信息不全的记录。

#### （三）弱证据或无效证据

原则上不能单独作为上黑依据：

1. 明显裁剪、拼接、伪造的截图；
2. 仅有主观描述，无对应证据；
3. 无法判断时间、对象、场景的材料；
4. 与举报事项关联性过弱的内容。

### 第七条 证据不足处理规则

证据不足时，审核人员可作以下处理：

1. 驳回举报；
2. 标记待补证；
3. 转入观察记录；
4. 仅作风险备注，不直接上黑；
5. 提交二次复核。

## 第三章 审核维度

### 第八条 核心审核维度

审核时，原则上从以下五个维度综合判断：

#### 1. 行为性质

主要判断行为本身属于什么类型，例如：

1. 轻度违规；
2. 骚扰；
3. 恶意引战；
4. 攻击辱骂；
5. 跨群扰乱；
6. 威胁、诈骗、现实骚扰等高危行为。

#### 2. 影响范围

主要判断行为影响了多少对象、多少场景：

1. 仅影响单一场景；
2. 影响多名用户；
3. 影响多个群聊或多个管理场景；
4. 已扩散到线下或现实层面。

#### 3. 持续性与重复性

主要判断行为是否偶发，还是多次、反复、持续发生：

1. 单次偶发；
2. 短期内重复；
3. 多次警告后仍继续；
4. 封禁后绕过继续实施。

#### 4. 主观恶意

主要判断当事人是否存在明显故意：

1. 无明显恶意，可能系冲动或误解；
2. 存在一定故意；
3. 明显故意挑衅、报复、绕规则；
4. 恶意持续、目的明确。

#### 5. 风险延续性

主要判断后续风险是否仍然存在：

1. 行为已停止，风险较低；
2. 仍有重复发生可能；
3. 已表现出报复、转移、扩散倾向；
4. 已构成持续高风险对象。

## 第四章 判级标准

### 第九条 一级（L1）判定标准

符合以下多数特征的，可判定为一级：

1. 违规事实较明确，但情节较轻；
2. 影响范围有限；
3. 对他人干扰轻微；
4. 多为一般群规问题；
5. 无明显持续性或高恶意；
6. 经提醒后有停止倾向。

典型情形：

1. 轻度刷屏；
2. 轻度引战；
3. 轻度影响群体验的言论；
4. 轻微违规宣传、发无关内容；
5. 初次违规且未造成明显后果。

### 第十条 二级（L2）判定标准

符合以下多数特征的，可判定为二级：

1. 违规已对他人形成明显干扰；
2. 存在重复性或规避性；
3. 已影响群秩序或用户体验；
4. 经提醒、警告后仍继续；
5. 主观恶意明显高于一般违规。

典型情形：

1. 持续私聊骚扰；
2. 多次辱骂、引战、恶意挑衅；
3. 反复刷屏、破坏正常讨论；
4. 利用规则漏洞持续干扰；
5. 多次被举报且核验属实。

### 第十一条 三级（L3）判定标准

符合以下多数特征的，可判定为三级：

1. 已影响多个群聊、多个用户或多个管理场景；
2. 行为具有持续性、扩散性；
3. 普通限制措施已难以阻断；
4. 被处理后仍有规避风控行为；
5. 对他人造成明显心理压力或持续困扰。

典型情形：

1. 跨群骚扰；
2. 对多名用户进行长时间纠缠、辱骂或威胁；
3. 被封禁后换号继续；
4. 恶意组织挑事、煽动冲突；
5. 已造成多个群管理成本显著上升。

### 第十二条 四级（L4）判定标准

符合以下任一高危特征，原则上可进入四级审核：

1. 涉及现实安全威胁；
2. 涉及隐私泄露、人肉、线下骚扰；
3. 涉嫌诈骗、勒索、恶意冒充等严重风险行为；
4. 具有组织化、持续性、极强恶意；
5. 已达到必须采取最高等级风控措施的程度。

典型情形：

1. 威胁现实人身安全；
2. 恶意公开他人隐私；
3. 现实跟踪、线下骚扰；
4. 高强度恶意攻击且风险未解除；
5. 需考虑移交更高层处理的案件。

## 第五章 辅助评分规则

### 第十三条 评分目的

为减少审核尺度不统一问题，可采用辅助评分机制。
评分仅作为参考，不替代管理员最终判断。

### 第十四条 建议评分项

每起案件可按以下维度评分：

#### 1. 行为严重性

0分：无明确违规
1分：轻微违规
2分：一般违规
3分：明显干扰
4分：严重骚扰或攻击
5分：高危或现实风险

#### 2. 影响范围

0分：无明显影响
1分：单一对象
2分：少量用户
3分：多个用户
4分：多个群
5分：跨平台或现实扩散

#### 3. 持续性

0分：单次且已停止
1分：偶发
2分：短期重复
3分：反复出现
4分：持续进行
5分：处理后仍绕过继续

#### 4. 主观恶意

0分：无明显恶意
1分：轻微冲动
2分：一定故意
3分：明显故意
4分：报复性强
5分：恶意极高

#### 5. 历史记录

0分：无历史记录
1分：曾被提醒
2分：有L1记录
3分：有L2记录
4分：有L3记录或多次有效举报
5分：长期高风险对象

### 第十五条 参考分段

总分可作为辅助参考：

1. 0—6分：一般不建议上黑，或仅记录观察；
2. 7—10分：倾向L1；
3. 11—15分：倾向L2；
4. 16—20分：倾向L3；
5. 21分及以上：进入L4重点审核范围。

### 第十六条 特殊说明

即使总分未达到高等级，只要存在以下情形之一，仍可直接提级处理：

1. 现实威胁；
2. 隐私泄露；
3. 诈骗或伪装诈骗行为；
4. 明确的人肉、跟踪、线下骚扰；
5. 其他极端高风险情形。

## 第六章 审核流程细化

### 第十七条 举报审核流程

举报审核建议按以下顺序进行：

#### 第一步：确认材料是否可审

检查是否具备：

1. 举报对象；
2. 举报理由；
3. 基本证据；
4. 可识别的时间或场景。

材料明显不足的，可直接退回补证或驳回。

#### 第二步：确认事实是否成立

重点确认：

1. 被举报行为是否真实发生；
2. 是否存在断章取义；
3. 举报内容与证据是否一致；
4. 是否有伪造、诱导、挑衅性取证嫌疑。

#### 第三步：确认行为类型

先给行为分类，例如：

1. 群规违规；
2. 骚扰；
3. 人身攻击；
4. 恶意引战；
5. 跨群扰乱；
6. 高危行为。

#### 第四步：评估影响和风险

判断：

1. 影响了谁；
2. 影响了多少人；
3. 是否会继续；
4. 是否需要立即阻断。

#### 第五步：给出等级意见

结合准则与本细则，给出：

1. 不成立；
2. 成立但不建议上黑；
3. L1；
4. L2；
5. L3；
6. L4待复核。

#### 第六步：记录审核结论

应记录：

1. 审核结论；
2. 判级理由；
3. 主要证据；
4. 操作时间；
5. 审核人。

## 第七章 升级、降级与复核细则

### 第十八条 升级条件

存在以下情形之一的，可升级处理：

1. 短期内再次有效违规；
2. 影响对象扩大；
3. 出现绕过限制行为；
4. 申诉阶段发现更严重事实；
5. 原处罚明显偏轻。

### 第十九条 降级条件

存在以下情形之一的，可考虑降级：

1. 原证据支撑不足；
2. 部分事实不成立；
3. 行为虽成立但影响程度被高估；
4. 经观察期证明风险已下降。

### 第二十条 复核条件

以下案件应进入复核流程：

1. L3及以上案件；
2. 证据争议较大案件；
3. 举报人与被举报人说法严重冲突案件；
4. 申诉提供新证据案件；
5. 涉及现实风险或重大舆情风险案件。

### 第二十一条 四级案件特别规则

L4案件必须遵守以下要求：

1. 不得仅凭单一弱证据直接确认；
2. 原则上需两名管理员确认；
3. 必须形成书面或系统化审核说明；
4. 必须记录证据摘要与处理依据；
5. 有条件时优先保留原始记录而非仅保留截图。

## 第八章 申诉复审细则

### 第二十二条 申诉审查重点

申诉复审时，重点看以下内容：

1. 原处理依据是否充分；
2. 是否存在误判、错判、过度判级；
3. 申诉人是否提供新证据；
4. 申诉是否仅为情绪化否认；
5. 风险是否已经实际下降。

### 第二十三条 不予支持申诉的常见情形

存在以下情形之一的，原则上不支持：

1. 无新证据，仅重复否认；
2. 明显回避关键事实；
3. 继续骚扰审核人员或举报人；
4. 申诉内容与证据明显不符；
5. 在申诉期间再次违规。

### 第二十四条 可支持申诉的常见情形

存在以下情形之一的，可支持部分或全部申诉：

1. 原证据存在误解；
2. 截图断章取义；
3. 处理等级偏高；
4. 举报存在恶意构陷成分；
5. 实际对象、时间、上下文被错误识别。

## 第九章 常见案例参考

### 第二十五条 L1参考案例

1. 在群内多次发送无关内容，经提醒后停止；
2. 轻微阴阳怪气、影响交流体验，但未持续扩大；
3. 初次引战但很快停止，影响有限。

### 第二十六条 L2参考案例

1. 连续多次私聊打扰他人；
2. 在群里反复挑衅、辱骂，已引发多人不适；
3. 被管理员提醒后仍继续刷屏或扰乱秩序。

### 第二十七条 L3参考案例

1. 在多个群重复挑事；
2. 对多名用户持续骚扰、辱骂、纠缠；
3. 被踢后换号继续干扰；
4. 长期针对某人进行网络层面的纠缠骚扰。

### 第二十八条 L4参考案例

1. 威胁线下报复；
2. 曝光他人隐私并扩散；
3. 带有诈骗性质的恶意接触；
4. 现实骚扰、跟踪、人肉等高危行为。

---

## 第十章 审核人员注意事项

### 第二十九条 禁止简单按情绪审案

管理员不得因以下因素直接加重处理：

1. 个人好恶；
2. 私人矛盾；
3. 对当事人既往印象；
4. 舆论压力；
5. 单方片面叙述。

### 第三十条 审核说明建议格式

每次审核建议至少写出一句说明，格式可参考：

```text
结论：成立，判定 L2。
理由：存在连续骚扰行为，已对他人造成明显干扰；证据为完整聊天截图及后台记录，且用户曾被提醒后未停止。
```

### 第三十一条 审核结论类型

系统结论建议统一为以下几类：

1. 不成立；
2. 成立但不予上黑；
3. 成立，记观察；
4. 成立，L1；
5. 成立，L2；
6. 成立，L3，待复核；
7. 成立，L4，待双审；
8. 证据不足，待补证。

## 附：管理员快速判定口径

可以给管理员一个最简判断逻辑：

### 一看行为轻不轻

只是一般违规、轻微影响体验，先看L1。

### 二看有没有持续骚扰

已经明显干扰别人，而且不是一次性的，优先看L2。

### 三看是不是跨群、多人、长期

一旦影响多个群、多人，或者封了还绕过，优先看L3。

### 四看是否涉及现实风险或违法倾向

只要碰到隐私泄露、现实威胁、诈骗、人肉、线下骚扰，直接按L4思路走，至少进入双管理员复核。
